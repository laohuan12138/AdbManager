# 快速开始指南

## 1. 准备工作

### 安装 ADB 工具
如果还没有安装 ADB，请按照以下步骤：

**Windows:**
1. 下载 Android SDK Platform-Tools
2. 解压到任意目录（例如：C:\adb）
3. 将该目录添加到系统环境变量 PATH 中
4. 打开命令提示符，运行 `adb version` 验证安装

**验证 ADB 是否可用:**
```bash
adb version
```

应该显示类似：
```
Android Debug Bridge version 1.0.41
```

## 2. 启动程序

### 方式一：直接运行（开发模式）
```bash
go run main.go
```

### 方式二：编译后运行
```bash
# Windows
go build -o adbmanager.exe .
adbmanager.exe

# Linux/macOS
go build -o adbmanager .
./adbmanager
```

## 3. 连接设备

### USB 连接
1. 用 USB 线连接 Android 设备到电脑
2. 在设备上启用"USB 调试"模式
   - 设置 → 关于手机 → 连续点击"版本号"7次启用开发者选项
   - 设置 → 开发者选项 → 启用"USB 调试"
3. 在程序的"设备管理"标签页点击"刷新设备列表"
4. 选择设备即可开始使用

### 无线连接
1. 确保设备和电脑在同一局域网
2. 在设备上启用"无线调试"（Android 11+）或使用以下命令：
   ```bash
   # 先通过 USB 连接
   adb tcpip 5555
   # 断开 USB，然后连接
   adb connect 设备IP:5555
   ```
3. 在程序的"设备管理"标签页输入 IP:PORT
4. 点击"连接"按钮

## 4. 功能使用示例

### 4.1 执行命令
1. 在"设备管理"标签页选择设备
2. 切换到"命令执行"标签页
3. 输入命令，例如：`ls /sdcard`
4. 点击"执行命令"

或使用快捷命令按钮直接执行常用命令。

### 4.2 浏览文件
1. 选择设备
2. 切换到"文件管理"标签页
3. 默认显示 /sdcard 目录
4. 双击目录可进入，选择文件可进行下载、删除等操作

### 4.3 管理应用
1. 选择设备
2. 切换到"应用管理"标签页
3. 点击"刷新应用列表"
4. 选择应用后可进行安装、卸载、启动、停止等操作

### 4.4 信息采集
1. 选择设备
2. 切换到"信息采集"标签页
3. 点击相应按钮采集需要的信息
4. 或点击"全部采集"一次性采集所有信息

### 4.5 批量操作
1. 切换到"批量操作"标签页
2. 在左侧添加目标设备 IP:PORT
3. 或点击"从文件导入"批量导入
4. 点击"批量连接"连接所有设备
5. 在右侧刷新设备列表并选择要操作的设备
6. 执行批量命令、安装、截屏等操作

### 批量导入示例文件格式 (targets.txt)
```
192.168.1.100:5555
192.168.1.101:5555
192.168.1.102:5555
127.0.0.1:5555
```

## 5. 常用场景

### 场景1：快速截图
1. 选择设备
2. 切换到"文件管理"标签页
3. 点击"截屏"按钮
4. 选择保存位置

### 场景2：批量安装应用
1. 在"批量操作"中导入所有设备
2. 批量连接设备
3. 选择要安装的设备
4. 点击"批量安装APK"
5. 选择 APK 文件

### 场景3：获取设备信息
1. 选择设备
2. 切换到"设备信息"标签页
3. 点击"获取基本信息"
4. 或点击其他按钮获取详细信息

### 场景4：查找敏感信息
1. 选择设备
2. 切换到"敏感信息"标签页
3. 点击"扫描配置文件"或其他扫描按钮
4. 等待扫描完成查看结果

## 6. 注意事项

### 权限问题
- 某些功能需要设备授予相应权限
- 访问联系人、短信需要相应的读取权限
- WiFi 密码获取可能需要 Root 权限
- 敏感信息扫描某些路径需要 Root 权限

### 设备兼容性
- Android 4.4+ 系统均可使用基本功能
- 某些功能可能因 Android 版本不同而有差异
- 建议使用 Android 6.0+ 以获得最佳体验

### 网络要求
- 无线连接需要设备和电脑在同一局域网
- 批量操作建议在稳定的网络环境下进行
- 大文件传输时注意网络速度

### 安全提示
- 本工具为渗透测试和设备管理工具
- 仅在授权的设备上使用
- 采集的敏感信息请妥善保管
- 不要在公共网络下使用无线连接

## 7. 故障排除

### 问题：无法检测到设备
**解决方案：**
- 检查 USB 调试是否启用
- 尝试更换 USB 线或接口
- 在命令行运行 `adb devices` 检查
- 重启 ADB 服务：`adb kill-server` 然后 `adb start-server`

### 问题：无线连接失败
**解决方案：**
- 确保设备和电脑在同一网络
- 检查防火墙设置
- 确认设备 IP 地址正确
- 尝试先 USB 连接后再启用无线模式

### 问题：命令执行失败
**解决方案：**
- 检查命令语法是否正确
- 某些命令需要 Root 权限
- 确认设备已正确连接
- 查看错误信息了解具体原因

### 问题：文件访问被拒绝
**解决方案：**
- 某些系统目录需要 Root 权限
- 尝试访问 /sdcard 等公共目录
- 检查应用是否有文件访问权限
- 在设备上授予必要权限

## 8. 高级技巧

### 技巧1：使用自定义命令
在"命令执行"标签页可以执行任何 shell 命令，例如：
```bash
# 查看 CPU 信息
cat /proc/cpuinfo

# 查看正在运行的应用
ps -A | grep -i 应用包名

# 清除应用缓存
pm clear 应用包名
```

### 技巧2：导出批量操作结果
批量操作的结果会显示在界面上，可以复制保存。

### 技巧3：组合使用功能
1. 先用"信息采集"获取设备详情
2. 再用"敏感信息"扫描配置文件
3. 使用"文件管理"下载感兴趣的文件
4. 最后用"应用管理"清理或测试应用
